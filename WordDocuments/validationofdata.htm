Validation of Data

Data must be validated before being posted to the GL to ensure that it is complete and correct. Before ledger entries are posted to the GL, a transaction validation process is run to ensure that bad data, such as transactions applied to non-existent object codes or accounts, are not posted to the ledger.

Financial transaction documents generate pending ledger entries automatically. The KFS validates transactions created via e-docs in various ways, using the Chart of Accounts and business rules to ensure that the entries generated are complete and without errors. Even so, these transactions undergo the same 'Scrubber' validation as entries from other sources.
