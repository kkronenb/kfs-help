Business Rules

The following business rules apply to recurring cash transfer transactions.

 For the Edit Recurring Cash Transfer tab and its fields:

•      The KEMID must exist and must not be closed.

•      Only two transaction document types are allowed—Endowment Cash Transfer (ECT) and Endowment to General Ledger Cash Transfer (EGLT).

•      If the transaction document type is ECT, there can be associated target records only in the KEMID Transfer Target tab. There must be at least one active record for an active recurring transfer.

•      If the transaction document type is EGLT, there can be associated target records only in the GL Transfer Target tab. There must be at least one active record for an active recurring transfer.

•      The Etran code used in the Source Etran Code field must have an Etran type code of 'I' (Income) or 'E' (Expense).

•      The Etran code used must have an appropriately identified General Ledger object code record—one that matches the chart for the KEMID-associated General Ledger account.

o  If the source is income, the chart must match the chart of the active KEMID-GL Link record for income.

o  If the source is principal, the chart must match the chart of the active KEMID-GL Link record for principal.

•      The frequency code you specify must exist in the system.

•      If you specify a date in the Next Process Date field, it must be a valid date for the frequency entered.

•      If you do not enter a value in the Next Process Date field, when you save or submit the document the system automatically calculates and enters the next expected date for processing.

•      If you specify a date in the Last Process Date field, it must be a valid date for the frequency entered.

•      If you do not enter a value in the Last Process Date field, when you save or submit the document the system automatically calculates and enters the most recent date on which processing should have occurred.

•      If you deactivate a source record, the system automatically deactivates all target records.

•      When you copy a recurring cash transfer, only the source data is copied. You must enter all target information.

•      You cannot change the transaction document type when editing a record.

For the Edit KEMID Transfer Target tab and its fields:

•      You may enter multiple KEMID targets for a transfer.

•      The transfer number will be the same as the number for the associated Cash Transfer record.

•      The sequence number for each KEMID target is assigned by the system sequentially.

•      Each KEMID specified must exist in the system and must not be closed.

•      The target Etran code you specify must have an Etran type code of 'I' (Income) or 'E' (Expense).

•      The target Etran code you specify must have an appropriately identified General Ledger object code record—one that matches the chart for the KEMID-associated General Ledger account.

o  If the target is income, the chart must match the chart of the active KEMID-GL Link record for income.

o  If the target is principal, the chart must match the chart of the active KEMID-GL Link record for principal.

•      If you enter a value in the Target Amount field, you cannot enter data into the Target Percent or Use Etran Code fields for this KEMID target.

•      If you enter a value in the Target Percent field, the total for all active target KEMID target records cannot exceed 100%.

•      If you enter values in both the Target Percent and Use Etran Code fields, the total for all active KEMID target records with this Use Etran Code value cannot exceed 100%.

For the General Ledger Transfer Target tab and its fields:

•      You may enter multiple GL account targets for a transfer.

•      The transfer number will be the same as the number for the associated Cash Transfer record.

•      The sequence number for each GL account target is assigned by the system sequentially.

•      Each chart specified must exist in the system and must not be closed.

•      Each account specified must exist in the system and must not be closed.

•      Each object code specified must have a type code equal to 'I' (Income) or 'E' (Expense).

•      If you specify a sub-account, it must exist in the system, must be active, and must be associated with the account and chart in combination.

•      Each object code specified must exist in the system, must be active, and must be associated with the fiscal year and account's chart.

•      If you specify a sub-object code, it must exist in the system, must be active, and must be associated with the object code, account, chart, and fiscal year in combination.

•      If you specify a project code, it must exist in the system and must be active.

•      If you enter a value in the Target Amount field, you cannot enter data into the Target Percent or Use Etran Code fields for this GL account target.

•      If you enter a value in the Target Percent field, the total for all active target GL account target records cannot exceed 100%.

•      If you enter values in both the Target Percent and Use Etran Code fields, the total for all active GL account target records with this Use Etran Code value cannot exceed 100%.
