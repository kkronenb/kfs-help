Security

 >  >

The Security submenu group provides access to screens in which you define criteria for document and accounting line security based on accounting line attributes.

     Modules that rely on organization-based module security—that is, the Budget Construction, Accounts Receivable, and Capital Assets modules—are  not affected by settings in these creens. In these modules, organization-based security always controls user access.

Security submenu group

Document

Description

Attribute

Defines security attributes that may be used to place restrictions on user access.

Definition

Defines security rules (definitions) that may be assigned to a security model or principal.

Model

Defines security models—that is, collections of security definitions having specific operators and values and assigns them to roles, groups, and/or principals.

Principal

Associates security models and individual security definitions with specific principals. May be used to define security exception rules for principals.

All KFS users may look up valid values in these documents, but only members of the KR-SYS Technical Administrator or KFS-SYS Manager roles may copy, edit and create new records via these documents. These documents do not route for approval.
